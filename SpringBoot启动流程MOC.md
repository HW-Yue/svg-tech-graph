# SpringBoot启动流程MOC

> **核心模型：** Spring Boot 启动不是一个动作，而是一串阶段化的容器编排：入口识别、环境准备、上下文创建、配置类解析、自动装配、Bean 创建、内嵌 Web 容器启动、启动后回调。

- [[SpringBoot启动流程]] — 先读这篇总览，再按阶段往下拆。

## 2. 环境准备
- [[Environment准备]] — 把配置文件、命令行参数、JVM 参数和系统环境变量合成最终配置视图。

## 3. 应用上下文
- [[ApplicationContext创建]] — 根据应用类型创建对应上下文实现。

## 4. refresh 主流程
- [[refresh流程]] — 容器真正生效的总开关。
- [[ConfigurationClassPostProcessor]] — 配置类解析、组件扫描和 `@Bean` 注册的核心处理器。

## 5. 自动装配
- [[自动装配原理]] — 根据 classpath、属性和容器状态决定哪些配置生效。

## 6. Bean 创建与生命周期
- [[Bean生命周期]] — Aware、初始化、销毁和回调的完整链路。

## 7. Web 容器
- [[内嵌Web容器启动]] — Tomcat / Jetty / Undertow 的创建和绑定端口。

## 8. 启动后回调
- [[启动后回调]] — 启动完成后的标准回调入口。

## 9. 这条线的总览笔记
- [[SpringBoot启动流程]] — 启动流程的详细过程说明，适合先读一遍建立直觉。

## 学习路径建议
1. 先看 `[[SpringBoot启动流程]]`，把整条链路先跑通。
2. 再按 `入口 -> 环境 -> 上下文 -> refresh -> 自动装配 -> Bean 生命周期 -> Web 容器 -> 回调` 的顺序补子笔记。
3. 如果要继续往下深挖，就把每个阶段都拆成一个原子笔记，MOC 只保留链接和关系。
